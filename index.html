NRF Chip Transfer Format

Element Value Description
Request newIdentity The sensor sends this package to Duality requesting a new sensor ID
Request sensorData The sensor sends this package to Duality informing it that the rest of the package will be sensor data
ID/sensor_id sensor_id Duality uses the sensor_id's to separate data from different sensors and to put them in appropriate databases. The only time the sensor_id is not an integer is when it does not yet have an identity, in which case it will be "new_sensor"
ID/type sensor_type A string containing the type of sensor, eg: "temperature", "distance".
Payload payload A datatype containing the information from the sensors. The format will change depending on the type of sensor

Examples:

{

    "Request": "sensor_data",

    "ID": {

        "sensor_id": "new_sensor",

        "type": "distance"

    }
